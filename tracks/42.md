# 42 공통 선행 트랙

이 트랙은 Frontend와 Backend의 필수 선행이다. 아래 카드를 위에서 아래로 진행하고, 각 카드의
`다음` 링크로만 이동한다. 공통 규칙과 중단·재개 방법은 [전체 학습 시작점](README.md)에서 한 번만
읽는다.

시작 전에 저장소를 clone한 Document Box에서 다음을 실행한다.

```sh
make preflight TRACK=42
```

`BLOCK`을 해결하고, 뒤 단계용 `WARN`은 해당 카드에 도착하기 전 해결한다. 모든 `learning/*` ref는
읽기 전용이다. Release tag는 baseline에만 사용하고, 실제 구현은
[Basis 안내의 우선순위](README.md#basis-안내의-우선순위)에 따라 practice 또는 current ledger
crosswalk에서 해소한 historical parent의 개인 `study/*` branch에 둔다. 아래 카드의 “현재 practice의
full 부모 commit”도 이 해소 규칙을 줄여 쓴 표현이다. 대표 항목의 결정 규칙과 선택 심화의 경계는
[공식 수행 범위](README.md#공식-수행-범위)를 따른다.

## 0. Linux/Git Foundations 선수 과정

- 이전 gate: [전체 지도](README.md#42-공통-선행)에서 42 시작을 선택한다.
- 성격: 별도 프로젝트나 release가 아닌 Central Notes의 필수 선수 과정이다. Retired `linux-admin`
  저장소를 clone하거나 과거 answer/practice를 소비하지 않는다.
- 읽는 순서: [Linux/Git Foundations](https://github.com/woopinbell/central-notes/blob/main/TRACK_SEQUENCE.md#stage-linux-foundation)의 handoff를 위에서 아래로 따른다.
- 완료 artifact: shell에서 file/process/permission을 관찰한 기록, 작은 Git branch에서 diff·commit·self-review를 수행한 기록, rollback과 검증 명령을 개인 원장에 남긴다.
- 완료 gate: Central 카드가 지정한 quick probe와 self-review checklist를 통과하고 결과를 자기 말로 설명한다.
- 다음: [c-foundation](#stage-c-foundation).

## 14. 42 통합 incident 평가

- 이전 gate: Linux/Git Foundations와 13개 프로젝트의 무자료 gate·완료 artifact 기록.

- 시작: starter와 contract만 새 작업 디렉터리에 복사하고 project source·answer·이전 제출을 닫는다.

- 완료 gate: 자동 checker와 사람 rubric을 통과하고 즉시 회상 checkpoint를 기록한다.
- 회상: [42 incident clock](https://github.com/woopinbell/central-notes/blob/main/assessments/recall-checkpoints.md#42-incident-clock)의 7일·30일 일정을 예약한다. 30일 전에도 확장 트랙을 시작할 수 있지만 42 mastery 확정은 30일 통과 뒤다.
- 분기: [Frontend 시작](frontend.md#stage-frontend-foundations-training), [Backend 시작](backend.md#stage-backend-foundations-training) 중 하나 또는 둘 다 선택한다.
- 선택 브리지: 42를 과거에 직접 완주했고 지원 준비를 앞당기려면
  [Frontend 지원 준비 브리지](frontend-fast-track.md#route-frontend-application-bridge)를 선택한다.
  이 경로도 위 42 전체 복습과 incident 즉시 gate 뒤에만 진입하며 Frontend mastery를 부여하지 않는다.
