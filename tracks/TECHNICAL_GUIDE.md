# 학습용 Git 기술 안내

이 문서는 기본 학습 절차에서 막혔을 때만 참고한다. 먼저 [학습 시작](README.md)의 쉬운 절차를 따른다.

## 용어

- **완성본 태그(annotated release tag):** 검증된 현재 완성본을 가리키는 고정된 Git tag다.
- **시작 커밋(parent commit):** 정답 변경이 아직 적용되지 않은, 연습문제를 시작할 코드 시점이다.
- **커밋 대응표(crosswalk):** 문서의 문제 ID와 실제 commit·parent·tree를 연결한 표다.
- **검토 전용(metadata-only):** 코드를 구현하지 않고 commit 정보와 source를 확인·설명하는 항목이다.
- **tree:** 한 commit에 저장된 파일 전체의 상태다.
- **worktree:** 같은 저장소의 다른 commit을 별도 디렉터리에 동시에 펼치는 Git 기능이다.

## 두 코드 시점을 나누는 이유

연습문제는 과거의 작은 변경 하나를 재현한다. 현재 완성본은 그 뒤의 모든 변경까지 포함한다. 따라서 두
검사를 한 디렉터리에서 섞지 않는다.

1. 문제에서 지정한 시작 커밋의 개인 branch에서는 그 문제가 요구한 당시 검사만 실행한다.
2. 별도 worktree의 완성본 태그에서는 카드의 현재 전체 검사를 실행한다.

현재 명령이 과거 코드에 없다고 해서 연습이 실패한 것은 아니다. 반대로 과거 검사만 통과했다고 현재
완성본 전체가 정상이라고 기록해서도 안 된다.

## 시작 커밋을 찾는 방법

가장 먼저 연습문제 파일의 `부모 commit` 또는 `시작 commit` 값을 사용한다.

```sh
git switch -c study/<프로젝트>-<문제-ID> <시작-커밋>
```

문제 파일에 값이 없다면 해설 본문을 열지 말고 해설 목록이 연결한 커밋 대응표에서 같은 문제 ID의
정보 행만 확인한다. 짧은 SHA는 다음처럼 전체 SHA로 바꿔 기록한다.

```sh
git rev-parse <짧은-SHA>^{commit}
git show -s --format=%T <전체-커밋-SHA>
```

문제 파일과 대응표 모두에 시작 값이 없거나 서로 다른 값을 가리키면 진행을 멈추고 문서 교정을
요청한다. 추측한 commit에서 시작하지 않는다.

### root commit 문제

부모가 없는 첫 commit을 재구현하는 문제라고 명시된 경우에만 빈 디렉터리 또는 orphan branch를 쓴다.

```sh
git switch --orphan study/<프로젝트>-<문제-ID>
```

### 검토 전용 항목

`metadata-only` 또는 `검토 전용` 항목은 구현 문제가 아니다. 가짜 branch나 실패를 만들지 않는다.
commit hash·parent·tree, 분류와 구현 문제에서 제외된 이유를 기록하고 source의 책임을 자기 말로
설명한다.

## 현재 완성본을 별도로 확인하기

```sh
git fetch --tags origin
git worktree add --detach ../<저장소>-현재-확인 <완성본-태그>
cd ../<저장소>-현재-확인
# 프로젝트 카드의 현재 완성본 검사 실행
cd -
git worktree remove ../<저장소>-현재-확인
```

검사 결과에는 실행 명령, 반환값, 중요한 출력과 실행 시각을 남긴다.

## 오래된 학습 안내와 충돌할 때

오래된 읽기 전용 문서에는 완성본 태그에서 개인 branch를 만들거나 모든 연습문제를 수행하라는 문구가
남아 있을 수 있다.

- 문제 수와 선택 방법은 현재 Document Box의 [필수 학습 범위](README.md#공식-수행-범위)를 따른다.
- 실제 구현 범위와 당시 검사는 선택한 연습문제 파일을 따른다.
- 구현 시작점은 문제 파일의 시작 커밋을 우선하고, 없을 때만 해설 목록의 대응표를 확인한다.
- 공개된 tag나 읽기 전용 learning branch를 이 문구 때문에 이동하거나 고치지 않는다.

## 문제가 생겼을 때 멈춰야 하는 경우

- 시작 커밋이 두 문서에서 서로 다르다.
- 필요한 parent나 tree를 저장소에서 찾을 수 없다.
- 연습문제가 요구한 공개 API와 source가 다르다.
- 현재 완성본 검사가 실제 source 문제로 실패한다.

이 경우 진행 기록에 정확한 경로·commit·명령·오류를 남기고 문서 또는 source 교정을 요청한다.
