# 학습 포트폴리오 — 읽기 안내 (document-box)

`~/Desktop/seungwoo`는 여러 독립 레포로 된 **학습 포트폴리오**다. 각 레포는 한 스택(예: Spring · React ·
Android · RAG)을 **직접 재구성하며** 익힌 기록이고, 그 과정을 정해진 형태의 문서로 남긴다. 이 README는
**코드를 처음 보는 독자**가 각 레포의 폴더가 무엇이고 어떻게 읽는지 한눈에 알도록 정리한다(룰·절차 정본은
맨 아래 §거버넌스 문서). 이 기준을 레포마다 복붙하면 겹치므로 **여기 한 곳**에 둔다.

## 레포 한 개의 구조

```
<repo>/
├─ notes/                  ① 스택 가이드(깊은 선행 학습) + reference-impl/(짝 실행체)
├─ src · app · *-main …    본 프로젝트 코드 (정답 산출물)
└─ docs/
   ├─ commits/  NNN.md      ③ 답지: 커밋별 완성 코드 + 설계 이유
   ├─ practice/ NNN.md      ④ 문제지: 답지에서 코드 본문을 비운 연습용
   ├─ notes/                ② 온디맨드 어휘(얕은 개념 노트)
   └─ reflection/           회고 · 실패 모드 · 변경 비용
```

## 네 가지 문서 — 역할과 품질 기준

| 폴더 | 역할 | 품질 기준 | 독자가 읽는 법 |
|---|---|---|---|
| **`notes/<stack>.md`** | 스택 **교과서** (깊은 선행 정독) | **4부 포맷**: 개념·멘탈모델 → 본 프로젝트 코드 분석 → 학습용 미니프로젝트 → 심화. 스택 1개 = 1노트, 짝 `reference-impl/`는 빌드 가능 | 그 레포가 여는 스택을 **처음 배운다면 여기부터** 깊게 |
| **`docs/commits/NNN.md`** (답지) | 커밋별 **완성 코드 + 왜** | 실제 git 커밋에 근거(추측 없음) · **완성 코드를 보여줌**(산문 설명만 X) · `## 검증` 실행 가능 · 끝에 L3/L2/L1 색인 | 구현이 *어떤 순서로* 쌓였는지, 각 결정의 이유를 따라 읽기 |
| **`docs/practice/NNN.md`** (문제지) | 답지에서 **코드 본문을 비운** 연습 | 계약(시그니처·타입·구조)은 노출, 본문은 `// TODO: 책임`, 답지와 **같은 논리 단위** | **답지를 덮고** 직접 채운 뒤 답지와 대조(재유도 학습) |
| **`docs/notes/<cat>/`** | **얕은 개념 어휘** (온디맨드) | 카테고리별 짧은 노트, **선택적**(없는 레포 있음) | 커밋을 따라가다 **막힐 때만** 펼치는 용어 사전 |

### `notes/` 와 `docs/notes/` 는 다르다 (가장 헷갈리는 지점)

같은 주제(예: `docker`)가 양쪽에 있을 수 있지만 **중복이 아니라 정반대 깊이**다:

- 루트 **`notes/`** = *깊게 선행 정독*하는 **교과서**(4부 포맷).
- **`docs/notes/`** = *얕게, 막힐 때만* 꺼내는 **곁노트(용어 사전)**.

둘 중 "더 최근 = 진짜"가 아니다 — **역할이 다른 두 계층**이다. 굳이 정본 하나를 꼽으면 루트 `notes/`(모든 레포가 가짐).

### L3 / L2 / L1 — 답지 끝의 복기 색인

각 답지(`docs/commits/NNN.md`) 끝에는 "이 커밋에서 무엇을 얼마나 내재화할지" 우선순위가 붙는다:

- **L3** — 문서 없이 **백지에서 설명·작도**할 수 있어야 하는 핵심(폭발 반경 큰 것: 동시성·트랜잭션·보안·돈 계산 등).
- **L2** — 소스·문서를 **펼쳐서** 흐름·예외·책임 분리를 설명할 수 있으면 충분.
- **L1** — 필요할 때 **읽어 파악**하면 됨.

## dual-form 한눈에

**`docs/commits/`(답지)가 정본**이고 **`docs/practice/`(문제지)는 거기서 파생**된다. 학습자는 문제지를 덮고
직접 풀어본 뒤 답지로 채점한다 — "다시 읽기"보다 **"다시 만들기"**가 핵심. (실체 있는 코드 커밋만 문제지를
가진다; 마커·순수 문서·순수 타입 정의 커밋은 답지만 둔다.)

## 캡스톤 · 멀티레포 폴더

대부분 레포는 독립적이지만, 여러 레포를 묶는 **캡스톤은 폴더**로 둔다 — 그 하위 각각이 독립 레포다:

- **`sportsbook/`** — 분산 백엔드(서비스 7 + gateway + orchestration) + 모바일 클라이언트.
- **`ai-capstone/`** — RAG 에이전트 등(rag-agent · eval-gate · reliability-gateway · orchestration · shared-contracts).

(이런 캡스톤 폴더는 더 생길 수 있다.)
