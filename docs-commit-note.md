# CLAUDE.md

이 파일은 `seungwoo/` 전체에 적용되는 **공통 룰만** 담는다. 레포별 상세 계획(시작 상태,
규모, 문서 매핑)은 여기 적지 않는다 — 각 레포를 맡은 세션이 그 자리에서 탐지해 스스로 세운다.

## 작업 공간과 작업 단위

- **작업 단위 = 독립 git 레포 하나.** 레포 목록은 박아두지 않는다. 탐지한다:
  `find . -maxdepth 3 -name .git -type d`의 부모 디렉터리가 작업 대상이다.
- **예외(멀티레포 폴더)**: `sportsbook/`·`ai-capstone/`은 레포가 아니라 **여러 개의 독립 레포를 담은
  폴더**다. 그 하위 각 레포가 각각 작업 단위다(상위 폴더를 한 덩어리로 다루지 않는다). 이런 *캡스톤 폴더*는
  앞으로 더 생길 수 있으니 폴더명을 박지 말고, `find`로 잡힌 하위 `.git`을 단위로 본다.
- **제외**: `document-box/`는 dual-form 대상이 아니다(프로그램 허브 — `SEQUENCE.md`·`LEARNING.md` 등).
  `plan-box/`도 비-git **기획 허브**라 대상이 아니다(`find`가 .git을 안 잡으므로 자동 제외).
  `launch-box/`는 **private git 레포**라 `find`에 잡히지만 출하·취업 실행 허브이므로 **명시적으로 제외**한다(document-box처럼 git이되 작업 단위 아님).

## 불변 규칙

독립 git 레포는 반드시 **자기 커밋 히스토리와 연동된 문서**(`docs/commits/`)를 가져야 한다.
문서가 없거나 빈 레포는 규칙 위반이며 메꿔야 할 대상이다.

## 목표 — dual-form

**실체 있는 구현 커밋**은 학습 문서를 두 형태로 갖춘다. (마커·순수 문서 커밋은 답지만 둔다 —
§표준 절차 3의 실체성 게이트.)

- `docs/commits/NNN.md` — **답지(정본)**: 완성 코드 + 설계 이유. 실제 git 커밋에 근거한다.
- `docs/practice/NNN.md` — **문제지(파생)**: 골격 + 책임 TODO + 검증. 답지에서 깎아낸 것.

원칙:

1. **답지가 정본, 문제지는 파생물.** 문제지를 손으로 창작하지 않는다. 답지가 바뀌면 재파생.
2. **답지는 실제 코드에 근거한다.** `git -C <repo> log/show`와 소스를 읽고 쓴다.
   **추측·환각 금지** — 근거를 못 찾으면 멈추고 보고한다.
3. **두 형태는 코드 본문과 "이유"만 다르다.** 헤더·개요·작업 순서·논리 단위 제목·검증은 공통.

## 작업 모델 — 레포 단위, 병렬, 자율 계획

- **한 레포 = 한 세션(또는 한 서브에이전트).** 그 세션이 자기 레포를 탐지하고 **맞춤 계획을
  스스로 세운다.** CLAUDE.md는 공통 룰만 준다. 티어·시작 상태·문서 매핑은 런타임 판단이다.
- 이 워크플로에서는 **서브에이전트 사용을 허용한다.** 여러 레포를 병렬로 돌릴 때 디스패처는
  레포당 에이전트 하나를 띄운다. 디스패치 프롬프트엔 (a) 대상 레포 경로, (b) "공통 룰은
  `document-box/CLAUDE.md`(→ `corpus-router.md`·`docs-commit-note.md`·`commit-policy.md`)를 읽고
  따르라", (c) "추측 금지, git으로 근거 확인"을 넣는다. (워크스페이스 루트엔 더 이상 auto-load
  CLAUDE.md가 없으니 명시적으로 읽힌다 — 레포 자체 CLAUDE.md가 있으면 그건 상속된다.)
- **레포 내 규모는 그 세션이 자기 문서 수로 판단한다**(`ls docs/commits/[0-9]*.md | wc -l`):
  - 작으면(≲15) 인라인으로 처리한다.
  - 크면(≳40) 자기 안에서 연속 범위(약 20개씩)로 서브에이전트를 분할하고, 동시 3~4개 한도를
    지키며, 끝날 때마다 드리프트 체크 + 표본 검수를 한다.

## 확장 작업 모델 — 완결된 레포에 후속 작업이 생길 때

경계 분리(commit-policy §날짜·배치)는 **phase 단위**다. 완결된 레포를 다시 여는 기준:

1. **문서만 고친다**(노트·답지·문제지 개선) → 현행대로 **메타 구간 커밋**(phase를 열지 않는다).
   누적되면 squash로 정리(트리 해시 보존 검증).
2. **기존 레포의 소스를 고치거나 보강한다**(결함 fix·관측/계약 정렬·기능 보강) → **새 phase**:
   구현 블록(fix/feat 커밋, 날짜는 현재 시점 이후 시간순) → 답지 **append**(직전 번호+1, 해당
   커밋 근거) → 그 phase 메타 블록(README phase 경계 명시, 필요한 문제지·노트 갱신). 소형
   fix(자명한 1~수 줄)는 실체성 게이트대로 답지만 둔다.
3. **독립적인 신규 능력 덩어리**(새 트랙·새 랩) → **새 레포**(캡스톤 폴더 하위 가능). 자기
   윈도우와 자기 docs를 가진다 — 기존 레포의 경계를 건드리지 않는 기본 경로.

답지의 `## 알려진 소스 결함` 섹션이 가리키던 결함을 phase가 수정하면, 그 섹션에 **수정 phase
커밋을 추기**한다(섹션 삭제가 아니라 이력 보존).

## 표준 절차 (레포를 맡은 세션이 따른다)

1. **시작 상태 판정** (`ls docs/commits/[0-9]*.md`, `grep -l 'TODO' docs/commits/*.md`, 답지 코드블록 수):
   - **문서 없음**(없거나 빔) → 불변 규칙 위반. Phase 0′.
   - **문제지형**(`// TODO:` 골격) → Phase 0.
   - **산문-only 답지**(코드 커밋인데 답지에 코드블록이 ~0 — 코드를 보여주지 않고 산문으로
     설명만) → Phase 0″. **"답지형"으로 착각 금지: TODO 없음 ≠ 코드 보여줌.**
   - **답지형**(실제 코드를 보여주는 완성 답지) → 바로 Phase 1.
2. **Phase 0 — 답지 정본화 (해당 상태만)**, `git -C <repo> log --oneline`으로 매핑한다.
   - **Phase 0′ (문서 없음)**: 히스토리 전체를 읽고 의미 있는 커밋마다 `commits/NNN.md`를
     신규 작성한다(merge·메타 커밋 제외, 번호 부여, `commits/README.md` 작성).
   - **Phase 0 (문제지형)**: `git show <hash>`로 실제 코드를 확인해 TODO 골격을
     완성 코드 + 이유로 채운다.
   - **Phase 0″ (산문-only 답지)**: `git show <hash>`의 실제 코드를 답지에 **보여주도록**
     채운다(시그니처+본문+이유). 산문 설명을 코드로 대체·보강해 small-shell 수준으로 만든 뒤
     파생한다(답지 충실도 게이트).
3. **Phase 1 — 문제지 파생 (실체성 게이트 적용)**: **실체 있는 커밋만** `commits/NNN.md` →
   `practice/NNN.md`로 파생한다(아래 §파생 규칙).
   - **실체 있는 커밋** = diff에 재유도할 구현이 있다(코드, Dockerfile/nginx/compose 같은
     config, 빌드·검증 스크립트). → 문제지를 만든다.
   - **순수 계약 패키지/커밋** = 로직 없는 DTO·type·schema·상수 정의(공유 타입 패키지,
     protocol 정의 등). → 기본 **스킵**(계약은 참조이고, 그 계약을 쓰는 구현 커밋이 문제지가
     된다). 단 재유도할 설계가 핵심인 schema 단위(검증 규칙·이벤트 union·도메인 스냅샷 모델)는
     **그 단위만 얇게 blank**한 문제지를 둔다. 순수 DTO 미러는 스킵.
   - **마커·순수 문서·이력 커밋** = `checkpoint-NN.md`만 추가, README·계획·회고 문서, 디렉터리
     의미 정리 등. → **답지만 두고 문제지를 만들지 않는다.** 재유도할 실체가 없는 문제지는
     공회전이다(템플릿 빈칸 채우기 ≠ 재구현).
   - 판단 신호: 그 커밋의 `## 검증`을 **실제 빌드/테스트/동작 확인으로 못 걸면 비구현**이다.
   파일명은 실체 있는 커밋만 1:1 미러링(`A`/`B` 분할 포함). 문제지가 하나라도 있으면
   `practice/README.md`를 한 장 둔다(사용법 + 어느 번호가 왜 빠졌는지 한 줄).
4. **Phase 2 — 답지 보강**: **실체 있는 답지**(문제지를 만든 것)에만 끝에 `## diff 반영 점검`
   (변경 파일 누락 0 검산)을 둔다. 마커·문서 답지는 1~소수 파일이라 커버리지가 자명하므로 생략.
   누락을 찾으면 답지부터 고치고 해당 문제지를 재파생한다.
5. **검증 — 실행 + 의미 셀프리뷰.**
   - **실행(필수):** 각 문서 `## 검증`의 명령을 **실제로 돌린다**. green이면 그대로. red면
     (a) 사전존재·설계상 실패(예: source-only validator가 `docs/` 금지, 별도 reference-impl
     타입 에러)면 그 사유를 해당 `## 검증`에 **명시**하고, (b) 실제 결함이면 멈추고 보고한다.
     **안 돌려보고 "통과"라고 적는 것 금지.**
   - **의미 셀프리뷰:** 답 없이 시작 가능한가 / 비운 칸마다 풀 문제가 실제로 있나 / 책임 줄이
     답을 누설 안 하나 / (B·C) 답지 코드가 전부 `git show`에 있나. 고위험(B 완성·C 저작)은
     **별도 새 에이전트**로 교차 리뷰한다(`###` 일치 같은 기계 체크는 보조일 뿐).
6. **마무리**: 검증을 통과하면 §커밋 & 푸시로 자동 커밋·푸시한다.

## 파생 규칙 (답지 → `practice/NNN.md`)

1. `#` 제목 / `## 개요` / `## 작업 순서`는 **그대로 복사**한다(문제 정의 역할).
2. `### 논리 단위` 제목은 **유지**한다.
3. 각 단위에서 **역할로** 비울지 노출할지 정한다(Level은 보조 신호):
   - **계약은 노출** — 함수 시그니처·프로토타입, 구조체 *이름*, 공개 API, include guard,
     자명한 보일러플레이트. 학습자가 시작점을 잃으면 안 된다.
   - **재유도 대상은 비움** — 함수 *본문*, 구조체 *필드 구성*, 알고리즘·제어흐름, 핵심 상수
     → `// TODO: <책임>`. 여기서 풀 문제가 생긴다.
   - 판정 한 줄: **"비우면 시작점을 잃나, 풀 문제가 생기나?"** 시작점 손실→노출, 문제 발생→비움.
4. **책임 줄**(`**책임:**`)은 "무엇/왜"만 주고 "어떻게"는 주지 않는다. 시그니처+책임만으로
   시도 가능하되 답이 베껴지지 않아야 한다. 빈 약속("X를 정의한다")도 답 누설(코드를 산문으로
   옮김)도 금지.
5. **`## 검증`**은 실제 빌드/테스트/관찰 가능한 동작이어야 한다. "파일에 X가 있나" 같은
   존재확인은 검증이 아니다 — 그건 비구현 신호다(§표준 절차 3 게이트).
6. 회고성 섹션(`## 결과`·`## 요약`·`## 다음 작업`·`## 기억/설명 Level`·`## diff 반영 점검`)은
   삭제. 최상단 배너: `> 생성물. 직접 수정 금지. 답: ../commits/NNN.md`.

**모범사례(온디맨드 참조):** 아래 **검증된 목록에 이름으로 적힌 쌍만** 참조한다. 목록에 없는
레포의 `docs/`는 보지 않는다 — 아직 작업 안 됐거나 검증 안 된 docs를 예시로 삼으면 나쁜 본을
베낀다. 에이전트가 "잘 된 것"을 찾아 다른 docs를 뒤지지 않는다. 새 쌍은 사람이 검증한 뒤
이 목록에 추가될 때만 모범사례가 된다.

검증된 모범사례 목록:

- **코드(C·계약 중심)** → `small-shell/docs/commits/002.md` + `practice/002.md`:
  `shell.h`를 논리 단위로 쪼갠 작업단위 분할 + 계약 노출 / 본문 비움 판단.
- (다른 유형은 아직 없음 — 생기면 여기에 이름으로 추가.)

대상 유형이 목록에 있으면 그 쌍을, 없으면 위 002 + 아래 교차유형 적용으로 푼다.

**예시는 베낄 틀이 아니라 원리의 사례다.** 가져갈 것은 §파생 규칙 3의 "역할로 비움/노출"
판단이지 C·헤더 관용구가 아니다. 아티팩트에 맞게 옮긴다:

- 코드 → 시그니처·프로토타입 노출, 본문 비움
- config(Dockerfile/nginx/compose) → 키·구조 노출, 값·핵심 디렉티브 비움
- 테스트 → 케이스 의도·이름 노출, assertion·입력 비움
- UI/문서형 → 골격·계약 노출, 로직·내용 비움

매몰 방지: 예시의 *모양*이 아니라 *판단*을 가져온다.

## 드리프트 방지 (일관성 게이트)

- `practice/NNN.md`에 코드를 손으로 채우지 않는다. 답지에서만 파생한다.
- 체크: `practice/NNN.md`의 `###` 제목 목록 == `commits/NNN.md`의 `###` 제목 목록.
  어긋나면 재파생 신호다.

## 커밋 & 푸시 (작업 완료 후 자동)

**형식·날짜 정본: [`document-box/commit-policy.md`](document-box/commit-policy.md).** 요약 —
영어 명령형 제목 `<type>(<scope?>): ...` + 한국어 `[근거]/[변경]/[검증]` 본문(trivial 면제),
AI 트레일러 없음. 날짜는 KST·author==committer·근무시간(09–21)·중복금지로 타임라인에 맞춘다
(history 재작성·force-push 허용 — synthetic 타임라인 repo).

검증을 통과하면 **묻지 않고** 커밋·푸시한다:

- 현재 브랜치(`main`)에 직접 커밋. 별도 브랜치/PR 없음.
- **바뀐 파일만**: dual-form은 `git -C <repo> add docs/commits docs/practice`. 전체 `-A` 금지.
- dual-form 스코프: 답지 `docs(commits)`, 문제지 `docs(practice)` — 둘 다면 커밋 분리.
- `git -C <repo> push`. 실패하면 멈추고 보고. 변경 없음/부분 실패면 커밋 안 함.
- 서브에이전트로 나눠도 **커밋·푸시는 레포 세션이 마지막에 한 번.**

## 교차 레포 문서 — `document-box`

여러 레포를 가로지르는 문서(드물다)는 `document-box/`에 둔다. 한 레포 히스토리 1:1이 아니라
종합 설명이므로 불변 규칙·dual-form에서 제외되는 참조용 문서다. 현재는 `SEQUENCE.md`·
`LEARNING.md`뿐이다. (`sportsbook/` 전체를 묶는 umbrella는 필요해지면 여기 둔다 — 현재 미정,
요구사항 아님.)

## 금지

- `practice/` 직접 손수정, 답지 추측 작성, `docs/notes/` 변경.
- 검증 실패·부분 완료 상태에서 커밋·푸시. `-A`로 무관한 변경까지 싸잡아 커밋.
