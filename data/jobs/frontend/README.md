# Frontend 지원 데이터

Frontend 채용 공고처럼 수명이 짧은 상태와 직무별 증거 조합을 소유합니다. 트랙 순서·완료 정의·Git
정책은 상위 정본에 남기고 이 폴더에 복제하지 않습니다.

## 현재 문서

- [`2026-07-14.md`](2026-07-14.md): 2026-07-14 KST의 다중 플랫폼 후보 검색과 공식 원문 확인으로
  선별한 정규직 공고, 고용형태, 보상 하한·proxy와 지원 등급
- [`EVIDENCE.md`](EVIDENCE.md): 공고 요구사항에 연결할 저장소·검증 근거, 금지할 주장과 제출 gate

## 기록 규칙

1. 회사 공식 페이지나 ATS에서 지원 버튼과 활성 상태를 확인합니다.
2. 핵심 목록에는 기간의 정함이 없는 정규직만 둡니다. 해외는 공식 ATS의 `Full time`과 한국
   employee 또는 EOR 고용 가능성을 함께 확인하며, 독립계약자 offer만 가능하거나 고용형태가
   미확정이면 보류합니다.
3. 국내 공고는 공고상 연봉 하한 4,000만원 이상, 한국 거주자가 지원 가능한 worldwide remote는
   USD 60,000 이상만 둡니다. 지역별 범위가 있으면 한국 적용 여부가 확인돼야 합니다.
4. 국내 숫자가 없어도 플랫폼·기업이 데이터에 따라 자동 갱신하는 `연봉 상위 n%` 태그, 또는
   4,000만원 이상 회사 평균과 함께 표시되는 연봉수준 태그가 있으면 `보상 proxy`로 조건부 포함합니다.
   태그와 회사 평균은 개별 offer 하한이 아니므로 기본연봉 4,000만원 이상을 채용 담당자에게 확인하기
   전에는 `보상 확인` gate를 통과하지 못합니다.
5. 성과급·스톡옵션·복지는 하한 계산에서 제외합니다. 자동 갱신 근거가 없는 제3자 평균연봉,
   `업계 최고`, `시장 최고`, `협의`만 있는 공고도 넣지 않습니다. 이 하한은 시장 평균 통계가 아니라
   지원 대상을 좁히는 운영 기준입니다.
6. 개인 프로젝트는 `배포·검증된 개인 프로젝트`로 쓰고 `실무`, `프로덕션 운영`, `팀 성과`로
   바꾸어 쓰지 않습니다.
7. `공고 확인 → 지원 자료 준비 → 제출 → 전형 결과`를 분리합니다. 제출 증빙 없이 제출 완료로
   표시하지 않습니다.
8. 상시채용도 제출 직전에 공개 HTML뿐 아니라 지원 가능 상태를 다시 확인합니다. 다른 출처가 충돌하면
   회사 공식 페이지가 우선이고, 해결되지 않으면 보류합니다.

## 현재 판정

2026-07-14 snapshot 선별은 완료했지만 제출 packet은 아직 준비되지 않았습니다. 프로젝트 저장소를
모두 private로 운영하는 것은 visibility 전환과 원격 전수 audit가 끝난 뒤 확정합니다.
`portfolio-site`에는 live URL, 연락처와 핵심 Frontend 사례가 없습니다. 최종 audit 뒤 필요한
저장소만 채용 담당자에게 제한적으로 초대하는 방식과 상세 중단 조건은
[`EVIDENCE.md`](EVIDENCE.md)를 따릅니다. 이 데이터를 기록한 것만으로 지원이 제출되지는 않습니다.
