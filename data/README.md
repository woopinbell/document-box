# 변동 운영 데이터

트랙과 Git 정책의 정본이 아니라 특정 시점의 외부 상태와 실행 판단을 기록합니다. 공고·시장 관측,
지원 근거처럼 시간이 지나면 달라지는 자료를 안정적인 `tracks/**`와 분리합니다.

## 색인

- [`jobs/frontend/`](jobs/frontend/): Frontend 정규직 중 보상 하한 또는 검증 가능한 보상 proxy를
  갖춘 공고 snapshot과 증거 matrix

## 갱신 규칙

1. 파일명이나 문서 머리말에 확인 날짜와 timezone을 남깁니다.
2. 외부 상태는 회사 공식 페이지나 ATS를 우선하고, 출처가 충돌하면 해결 전까지 보류합니다.
3. 연봉은 공고나 회사 공식 페이지에 수치가 있을 때만 통화, 적용 지역과 보상 범위를 함께 씁니다.
   직무별 snapshot의 숫자 하한은 국내 연 4,000만원, 한국에서 지원 가능한 worldwide remote는 연
   USD 60,000입니다.
4. 공고 숫자가 없어도 플랫폼·기업이 데이터에 따라 자동 갱신하는 `연봉 상위 n%` 태그, 또는
   4,000만원 이상 회사 평균과 함께 표시되는 연봉수준 태그는 `보상 proxy`로 조건부 포함할 수 있습니다.
   개별 offer의 4,000만원을 보장한다고 바꾸어 쓰지 않고 채용 담당자에게 기본연봉 하한을 확인합니다.
   `업계 최고`, `시장 최고`, `협의`나 출처 불명의 평균만으로는 포함하지 않습니다.
5. 위 금액은 시장 평균을 주장하는 통계가 아니라 이 데이터셋의 지원 선별선입니다. 성과급·스톡옵션·복지는
   하한 충족에 합산하지 않고, 특정 국가에만 적용되는 보상 범위는 한국 지원자의 하한 근거로 쓰지 않습니다.
6. 새 관측은 과거 snapshot을 덮어쓰지 않고 새 날짜 파일로 추가합니다. 명백한 기록 오류만 기존
   문서에서 교정하고 이유를 commit 본문에 남깁니다.
7. 이 데이터는 트랙 순서, release 완료, commit/ref 정책을 바꾸지 않습니다. 충돌하면 `tracks/**`와
   정책 정본이 우선합니다.
8. 제출 ID, 개인 연락처, 이력서 원문처럼 공개하면 안 되는 개인정보는 commit하지 않습니다.
9. 새 직무군은 해당 `jobs/<직무>/` 폴더와 위 색인의 한 줄을 같은 commit에 추가합니다. 직무군을
   순차 게시할 때는 먼저 게시한 폴더만 중간 tree의 색인에 두며, 아직 없는 다음 폴더를 미리 링크하지
   않습니다.
